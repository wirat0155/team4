ผมเขียนในฐานะที่อยู่ในตำแหน่ง QA นะครับ
(งานส่วนใหญ่คือการตรวจสอบเอกสาร ออกแบบ Test Scenario/Case/Script และ Integration Test)

1. เครื่องมือในการพัฒนาระบบ ( Tools )
          เครื่องมือส่วนใหญ่จะเป็นการนำไปใช้กับการทดสอบระบบ ซึ่งเราได้ใช้เครื่องมือที่ชื่อว่า Robot Framework ในการช่วย Automate Test จาก Test Case ต่างๆที่เราออกแบบ 
          ที่เราใช้ Robot Framework เพราะว่าเป็นเครื่องมือที่ถูกใช้สอนในรายวิชา 88833859 Software Testing and Quality Assurance และเพื่อเป็นการต่อยอด เราจึงนำมาใช้ในการ Automate Test และเห็นผลได้ว่าสามารถนำมาใช้ได้จริง

2. เทคนิค (Techniques)
          เทคนิคในการเลือก Test Scenario/Case ขึ้นอยู่กับปริมาณงานของ sprint นั้นๆ แต่หลักๆคือสุ่มหยิบมอดูลมาทำ จากนั้นมอดูลที่เลือกจะนำไป Integration Test แบบ manual หรือ automate(automate ที่ทำไปแล้วตอนนี้คือมอดูลเอเย่นต์) 
           เทคนิคการทำ manual นั้นเราจะทดสอบจากระบบจริงตั้งแต่ขั้นตอนแรกจนถึงขั้นตอนสุดท้าย หากพบจุดที่ทดสอบแล้วไม่ตรงตามกรณีหรือเจอข้อผิดพลาดก็จะทำการเช็ค รายงานในเอกสาร แล้วส่งแก้ เพื่อให้ทุกกรณีเป็นจริงตาม Test Scenario/Case
           เทคนิคการตรวจเอกสารจะเป็นการตรวจตั้งแต่ต้นจนจบหน้าของเอกสาร หากข้อผิดพลาดที่เล็กน้อยก็จะแก้ทันทีแต่หากมีมากเกินไปจะใส่ข้อเสนอแนะแล้วส่งให้คนที่รับผิดชอบเอกสารนั้นไปแก้ จากนั้นจะเช็คข้อผิดพลาดลงใน เอกสารchecklist
